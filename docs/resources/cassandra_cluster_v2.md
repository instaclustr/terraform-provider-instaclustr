# instaclustr_cassandra_cluster_v2 (Resource)
## Example Usage
```
resource "cassandra_cluster_v2" "example" {
  pci_compliance_mode = false
  data_centre {
    client_to_cluster_encryption = true
    cloud_provider = "AWS_VPC"
    continuous_backup = true
    name = "MyTestDataCentre1"
    network = "10.0.0.0/16"
    node_size = "CAS-PRD-m6g.large-120"
    number_of_nodes = 3
    private_ip_broadcast_for_discovery = true
    region = "US_EAST_1"
    replication_factor = 3
  }

  data_centre {
    client_to_cluster_encryption = true
    cloud_provider = "AWS_VPC"
    continuous_backup = true
    name = "MyTestDataCentre2"
    network = "10.1.0.0/16"
    node_size = "CAS-PRD-m6g.large-120"
    number_of_nodes = 3
    private_ip_broadcast_for_discovery = true
    region = "US_EAST_2"
    replication_factor = 3
  }

  spark {
    version = "<x.y.z>"
  }

  private_network_cluster = false
  cassandra_version = "<x.y.z>"
  lucene_enabled = true
  password_and_user_auth = true
  name = "MyTestCluster"
  two_factor_delete {
    confirmation_email = "test@email.com"
  }

  sla_tier = "PRODUCTION"
}
```
## Schema
### pci_compliance_mode<br>
<ins>Type</ins>: boolean, required, immutable<br>
<br>Creates a PCI compliant cluster, see [PCI Compliance](https://www.instaclustr.com/support/documentation/useful-information/pci-compliance/).
### two_factor_delete<br>
<ins>Type</ins>: block list, see [two_factor_delete](#nested--two_factor_delete) for nested schema, optional, updatable<br>
<ins>Constraints</ins>: maximum items: 1<br>
### default_user_password<br>
<ins>Type</ins>: string, read-only<br>
<br>Password of the default user created for the Cassandra cluster.
### data_centre<br>
<ins>Type</ins>: block list, see [data_centre](#nested--data_centre) for nested schema, required, updatable<br>
<ins>Constraints</ins>: minimum items: 1<br><br>List of data centre settings.
### current_cluster_operation_status<br>
<ins>Type</ins>: string, optional, updatable<br>
<ins>Constraints</ins>: allowed values: [ `NO_OPERATION`, `OPERATION_IN_PROGRESS`, `OPERATION_FAILED` ]<br><br>Indicates if the cluster is currently performing any restructuring operation such as being created or resized
### spark<br>
<ins>Type</ins>: block list, see [spark](#nested--spark) for nested schema, optional, immutable<br>
<ins>Constraints</ins>: maximum items: 1<br><br>Adds the specified version of Apache Spark to the Cassandra cluster.
### private_network_cluster<br>
<ins>Type</ins>: boolean, required, immutable<br>
<br>Creates the cluster with private network only, see [Private Network Clusters](https://www.instaclustr.com/support/documentation/useful-information/private-network-clusters/).
### cassandra_version<br>
<ins>Type</ins>: string, required, immutable<br>
<ins>Constraints</ins>: pattern: `[0-9]+\.[0-9]+\.[0-9]+`<br><br>Version of Cassandra to run on the cluster.
### lucene_enabled<br>
<ins>Type</ins>: boolean, required, immutable<br>
<br>Adds Apache Lucene to the Cassandra cluster.
### password_and_user_auth<br>
<ins>Type</ins>: boolean, required, immutable<br>
<br>Enables Password Authentication and User Authorization.
### name<br>
<ins>Type</ins>: string, required, immutable<br>
<ins>Constraints</ins>: pattern: `[a-zA-Z0-9][a-zA-Z0-9_-]*`<br><br>Name of the cluster.
### id<br>
<ins>Type</ins>: string, read-only<br>
<br>ID of the cluster.
### sla_tier<br>
<ins>Type</ins>: string, required, updatable<br>
<ins>Constraints</ins>: allowed values: [ `PRODUCTION`, `NON_PRODUCTION` ]<br><br>SLA Tier of the cluster. Non-production clusters may receive lower priority support and reduced SLAs. Production tier is not available when using Developer class nodes. See [SLA Tier](https://www.instaclustr.com/support/documentation/useful-information/sla-tier/) for more information.
### status<br>
<ins>Type</ins>: string, read-only<br>
<br>Status of the cluster.
<a id="nested--two_factor_delete"></a>
## Nested schema for `two_factor_delete`<br>

### confirmation_phone_number<br>
<ins>Type</ins>: string, optional, immutable<br>
<br>The phone number which will be contacted when the cluster is requested to be delete.
### confirmation_email<br>
<ins>Type</ins>: string, required, immutable<br>
<br>The email address which will be contacted when the cluster is requested to be deleted.
<a id="nested--aws_settings"></a>
## Nested schema for `aws_settings`<br>
AWS specific settings for the Data Centre. Cannot be provided with GCP or Azure settings.<br>
### ebs_encryption_key<br>
<ins>Type</ins>: string (uuid), optional, immutable<br>
<br>ID of a KMS encryption key to encrypt data on nodes. KMS encryption key must be set in Cluster Resources through the Instaclustr Console before provisioning an encrypted Data Centre.
### custom_virtual_network_id<br>
<ins>Type</ins>: string, optional, immutable<br>
<br>VPC ID into which the Data Centre will be provisioned. The Data Centre's network allocation must match the IPv4 CIDR block of the specified VPC.
<a id="nested--tag"></a>
## Nested schema for `tag`<br>
List of tags to apply to the Data Centre. Tags are metadata labels which  allow you to identify, categorize and filter clusters. This can be useful for grouping together clusters into applications, environments, or any category that you require.<br>
### value<br>
<ins>Type</ins>: string, required, immutable<br>
<br>Value of the tag to be added to the Data Centre.
### key<br>
<ins>Type</ins>: string, required, immutable<br>
<br>Key of the tag to be added to the Data Centre.
<a id="nested--gcp_settings"></a>
## Nested schema for `gcp_settings`<br>
GCP specific settings for the Data Centre. Cannot be provided with AWS or Azure settings.<br>
### custom_virtual_network_id<br>
<ins>Type</ins>: string, optional, immutable<br>
<br>Network name or a relative Network or Subnetwork URI e.g. projects/my-project/regions/us-central1/subnetworks/my-subnet. The Data Centre's network allocation must match the IPv4 CIDR block of the specified subnet.

<ins>Type</ins>: string, read-only<br>
<br>Rack name in which the node is located.
### node_size<br>
<ins>Type</ins>: string, read-only<br>
<br>Size of the node.
### private_address<br>
<ins>Type</ins>: string, read-only<br>
<br>Private IP address of the node.
### public_address<br>
<ins>Type</ins>: string, read-only<br>
<br>Public IP address of the node.
### id<br>
<ins>Type</ins>: string, read-only<br>
<br>ID of the node.
### status<br>
<ins>Type</ins>: string, read-only<br>
<br>Provisioning status of the node.
<a id="nested--azure_settings"></a>
## Nested schema for `azure_settings`<br>
Azure specific settings for the Data Centre. Cannot be provided with AWS or GCP settings.<br>
### resource_group<br>
<ins>Type</ins>: string, optional, immutable<br>
<br>The name of the Azure Resource Group into which the Data Centre will be provisioned.
<a id="nested--data_centre"></a>
## Nested schema for `data_centre`<br>
List of data centre settings.<br>
### continuous_backup<br>
<ins>Type</ins>: boolean, required, immutable<br>
<br>Enables commitlog backups and increases the frequency of the default snapshot backups.
### replication_factor<br>
<ins>Type</ins>: integer, required, immutable<br>
<br>Number of racks to use when allocating nodes.
### aws_settings<br>
<ins>Type</ins>: block list, see [aws_settings](#nested--aws_settings) for nested schema, optional, immutable<br>
<ins>Constraints</ins>: maximum items: 1<br><br>AWS specific settings for the Data Centre. Cannot be provided with GCP or Azure settings.
### number_of_nodes<br>
<ins>Type</ins>: integer, required, updatable<br>
<br>Total number of nodes in the Data Centre. Must be a multiple of `replicationFactor`.
### private_ip_broadcast_for_discovery<br>
<ins>Type</ins>: boolean, required, immutable<br>
<br>Enables broadcast of private IPs for auto-discovery.
### network<br>
<ins>Type</ins>: string, required, immutable<br>
<br>The private network address block for the Data Centre specified using CIDR address notation. The network must have a prefix length between `/12` and `/22` and must be part of a private address space.
### tag<br>
<ins>Type</ins>: block list, see [tag](#nested--tag) for nested schema, optional, immutable<br>
<br>List of tags to apply to the Data Centre. Tags are metadata labels which  allow you to identify, categorize and filter clusters. This can be useful for grouping together clusters into applications, environments, or any category that you require.
### gcp_settings<br>
<ins>Type</ins>: block list, see [gcp_settings](#nested--gcp_settings) for nested schema, optional, immutable<br>
<ins>Constraints</ins>: maximum items: 1<br><br>GCP specific settings for the Data Centre. Cannot be provided with AWS or Azure settings.
### client_to_cluster_encryption<br>
<ins>Type</ins>: boolean, required, immutable<br>
<br>Enables Client ⇄ Node Encryption.
